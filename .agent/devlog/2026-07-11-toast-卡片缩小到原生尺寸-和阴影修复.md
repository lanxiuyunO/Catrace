# 2026-07-11 Toast 卡片缩小到原生尺寸 + 阴影修复

## Session goal

Toast 卡片太大块，宽度/字体/内部留白都超标，要缩小到接近 Windows 11 原生通知（参考 Snipaste 更新提示）的大小；随后修复阴影显示不全。

## Completed

- 全面收紧尺寸：窗口宽 360、卡片最小高 128、padding 12、标题 14px、正文 13px、按钮 28px 高、计时球 84px；root 留白 16
- 宽度回退过一次：先收到 320，用户反馈比原生窄，改回 360（卡片宽跟随窗口，固定 320px 宽卡片已废弃）
- 阴影修复三轮定位：加深为双层阴影 → 左右被 `.toast-stack` 的 overflow 裁剪 → 顶部也被裁；最终用「四边 padding 借出血 + 负 margin + 窗口高度计算减 32px」根治
- 顺手清理 `EyeToastCard.vue` 里重复定义的 `.body-text` 块
- `pnpm vue-tsc --noEmit` 与 `cargo check` 均通过

## 关键经验

透明无边框窗口里，任何带 overflow 的祖先都是裁剪区；卡片贴边布局下阴影必被裁。固定套路：容器 padding 借出血 + 负 margin 拉回 + 高度测量时减去 padding。
